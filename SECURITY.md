# Security Policy

## Scope

SimuTrace is a read-only, client-side browser application. It never requests wallet signatures, never submits transactions, and never sends private keys or secret keys to any server.

All RPC calls are made to public Soroban RPC endpoints using the user's browser. No data is sent to any server controlled by this project.

## Reporting a Vulnerability

If you discover a security issue in SimuTrace — such as unintended exposure of sensitive data, a vector that could lead to unauthorized transaction signing, or a dependency vulnerability — please do not open a public issue.

Report it privately to the maintainers at the email address listed in the GitHub repository's security advisory tool.

We will acknowledge receipt within 48 hours and provide a timeline for a fix.

## Responsible Disclosure

We ask that you give us reasonable time to fix and release a patch before disclosing the issue publicly.
